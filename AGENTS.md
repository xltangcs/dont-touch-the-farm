# AGENTS.md

## Project overview

- Godot 4.6 project (config_version=5), Forward Plus renderer, D3D12 on Windows.
- 2D game (Node2D scenes) though 3D physics engine (Jolt) is configured.
- Scripting: GDScript only.

## Key files & entrypoints

- Main scene: `scenes/main_scene.tscn` (uid://b1aewbxsnqiev)
- Background scene (tilemap): `scenes/background_scene.tscn`
- Tileset: `tilesets/background.tres` (water source 1, grass source 3, tilled dirt source 4)
- Only script so far: `scenes/water.gd` — extends `TileMapLayer`, marked `@tool` (runs in editor), procedurally fills water tiles from viewport size on `_ready`

## Godot-specific conventions

- **`.import` files must be committed** alongside their source assets. They are the canonical import config.
- **`.uid` files** are auto-generated by the editor for each `.gd` script. Commit them.
- **`.godot/`** is gitignored (editor cache/metadata). Do not modify files inside `.godot/` directly.
- When adding new `.gd` scripts, the editor creates a companion `.uid` file automatically. If you create a `.gd` file by hand, open it in the editor so the UID is generated before committing.
- Scene/resource UIDs in `.tscn`/`.tres` files look like `uid://b1aewbxsnqiev` — these are the canonical refs. File paths are fallback. Use the editor to move/rename resources so UIDs are preserved.

## Asset organization

- `assets/game/characters/` — character spritesheets
- `assets/game/objects/` — furniture, tools, plants, items
- `assets/game/tilesets/` — tile source textures (grass, water, dirt, fences, houses, etc.)
- `tilesets/` — TileSet resources (`.tres`) that assemble tile sources

## Editor quirks

- `@tool` scripts (like `water.gd`) execute in the editor. Be careful with side effects — `water.gd` calls `fill_with_tiles()` → `clear()` on every `_ready`, which wipes manual tile placements.
- Rendering driver is set explicitly to `d3d12` for Windows. On other platforms Godot will fall back to its default.
